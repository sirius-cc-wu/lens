# Risk List

| ID | Risk | Type | Likelihood | Impact | Mitigation and evidence needed |
|---|---|---|---|---|---|
| `R-01` | A PlantUML server may expose diagram source, become unavailable, rate-limit requests, or change response behavior. | Technical / adoption | Medium | High | E1 validated live, invalid, unavailable, and delayed server responses; C1 and C2 use a controlled server for deterministic browser evidence. ADR-017 and C7 fix one destination per session, support a private or self-hosted server through `LENS_PLANTUML_SERVER`, forbid fallback after a configured-server failure, and retain the 10-second timeout, 2 MiB response limit, visible source, and per-diagram retry. C7 adds server normalization, removed-route, CLI, and browser evidence. Public-service availability and disclosure remain residual risks for zero-configuration sessions. |
| `R-02` | Unsanitized Markdown or SVG can execute unsafe browser content. | Technical / security | Medium | High | E1 escapes raw Markdown HTML, serves PlantUML SVG only as an image, and sets a restrictive content security policy. P10 parses only leading YAML frontmatter and escapes scalar, nested, and unknown metadata values before inserting them into the page; malformed YAML receives a fixed correction message. Expand malicious-content fixtures and browser testing before construction broadens the parser surface. |
| `R-03` | File and directory resolution can expose files outside the requested repository. | Technical / security | Low | High | E2 canonicalizes the document root, excludes discovered symbolic links and hidden entries, and serves only known document identifiers. Traversal and symlink behavior are tested. C2 verifies in a browser that a hidden Markdown file receives a Lens guidance page instead of its content. C3 verifies that the navigation pane lists the discovered documents, marks the current one, filters only that delivered list, and omits a hidden document and its source. P6 applies the same discovery and routing boundary to visible `.puml` files. C6 verifies that hiding the pane persists only as browser presentation state while a known document route still displays its authorized content and navigation marker after restoration. Large-repository discovery limits remain a residual risk. |
| `R-04` | "Display the codebase's code and document" is too ambiguous to estimate or scope. | Product | Low | Medium | ADR-004 defines V1 as documentation-only. Any source-code browsing requires a later product decision and use case. |
| `R-06` | Browser launch and process lifetime vary across desktop, container, and headless environments. | Technical | Medium | Medium | P7 tests the Linux `xdg-open`, macOS `open`, and Windows `cmd /C start` command constructions from one platform-neutral unit test. Each platform retains the manual loopback URL fallback; native-release and desktop-environment evidence remain part of release readiness. |
| `R-07` | A standalone distribution could be harder to install than an editor plugin. | Schedule / adoption | Medium | Medium | P2 adds a target-bound archive with a SHA-256 checksum, README, and license so users do not need a Rust toolchain. P7 generalizes the archive to native Linux, macOS, and Windows targets, preserves `lens.exe` on Windows, and retains the no-overwrite and extraction checks. P8 builds those artifacts on native tag-release runners and publishes them only after all three jobs succeed; native desktop and downloaded-archive testing remain release evidence. |
| `R-08` | Automatic refresh could rescan or expose files outside the authorized document set, replace readable content during a partial save, or trigger browser reload loops. | Technical / security | Medium | High | ADR-007 and C4 poll only canonical paths fixed at session creation, retain the last successful rendering on failed reads, and expose a revision-only current-document query. Unit tests cover changed and missing known files plus an unknown revision path; `BTE-01` saves the displayed document and verifies automatic refresh without `page.reload()`. Polling large document sets remains a residual performance risk. |
| `R-09` | A complete navigation catalog and client-side filter can make document responses and browser work impractical for large documentation trees, while an unrestricted search can weaken the fixed authorization boundary. | Technical / performance / security | Medium | High | ADR-008 and C5 use an immutable, session-built identifier index, explicit form submissions, 256-byte queries, and 50-item pages. Unit tests cover the cap, matching, limit, and invalid-page behavior; `BTE-01` submits a search and follows page two with JavaScript disabled against 51 matching documents. Matching cost on unusually large sets remains a residual measurement risk. |
| `R-10` | Removing renderer modes can break CLI and Rust API consumers and leave users who relied on local-command or disabled rendering without an equivalent workflow. | Product / compatibility | Medium | High | C7 removes `--renderer`, `RendererMode`, and the second `serve` argument together. CLI and type-bound tests verify both breaking surfaces; README migration guidance directs former local users to a private or self-hosted server and states that there is no no-rendering replacement. Pending release notes identify the CLI and library break before release. |

`R-01` through `R-04` are architectural and product risks that drive the first
elaboration iteration.
