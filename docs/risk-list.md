# Risk List

| ID | Risk | Type | Likelihood | Impact | Mitigation and evidence needed |
|---|---|---|---|---|---|
| `R-01` | The public PlantUML server may be unavailable, rate-limit requests, or change response behavior. | Technical / adoption | Medium | High | E1 validated a live SVG response plus valid, invalid, unavailable, and delayed mocked responses. C1 and C2 add browser-level success and failure paths against a controlled renderer, so external availability cannot make those regression tests fail. P1 adds a local-command and disabled-rendering session choice; P5 adds visible renderer status, per-diagram retry, and a session disable control. The local path retains a 10-second timeout and 2 MiB output limit. Public-service availability remains a residual risk only for the default public mode. |
| `R-02` | Unsanitized Markdown or SVG can execute unsafe browser content. | Technical / security | Medium | High | E1 escapes raw Markdown HTML, serves PlantUML SVG only as an image, and sets a restrictive content security policy. Expand malicious-content fixtures and browser testing before construction broadens the parser surface. |
| `R-03` | File and directory resolution can expose files outside the requested repository. | Technical / security | Low | High | E2 canonicalizes the document root, excludes discovered symbolic links and hidden entries, and serves only known document identifiers. Traversal and symlink behavior are tested. C2 verifies in a browser that a hidden Markdown file receives a Lens guidance page instead of its content. C3 verifies that the navigation pane lists the discovered documents, marks the current one, filters only that delivered list, and omits a hidden document and its source. Large-repository discovery limits remain a residual risk. |
| `R-04` | "Display the codebase's code and document" is too ambiguous to estimate or scope. | Product | Low | Medium | ADR-004 defines V1 as documentation-only. Any source-code browsing requires a later product decision and use case. |
| `R-06` | Browser launch and process lifetime vary across desktop, container, and headless environments. | Technical | Medium | Medium | V1 supports Linux `xdg-open` with a manual URL fallback. Validate the release behavior in the V1 readiness check. |
| `R-07` | A standalone distribution could be harder to install than an editor plugin. | Schedule / adoption | Medium | Medium | P2 adds a target-bound Linux archive with a SHA-256 checksum, README, and license so users do not need a Rust toolchain. The packaging command refuses overwrite and is checked by extracting a fresh archive; tagged publication remains proposal 8. |
| `R-08` | Automatic refresh could rescan or expose files outside the authorized document set, replace readable content during a partial save, or trigger browser reload loops. | Technical / security | Medium | High | ADR-007 and C4 poll only canonical paths fixed at session creation, retain the last successful rendering on failed reads, and expose a revision-only current-document query. Unit tests cover changed and missing known files plus an unknown revision path; `BTE-01` saves the displayed document and verifies automatic refresh without `page.reload()`. Polling large document sets remains a residual performance risk. |

`R-01` through `R-04` are architectural and product risks that drive the first
elaboration iteration.
